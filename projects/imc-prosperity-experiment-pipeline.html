IMC Prosperity Experiment Pipeline

A workflow I built during the IMC Prosperity trading competition to automate submissions, capture results, and compare strategy runs more easily.

GitHub repository IMC Prosperity website

Whats This?

IMC Prosperity is a market-making and algorithmic trading competition where teams design trading bots for simulated markets, interpret order-book behavior, and adjust strategies as new products and rules appear across rounds.

During the competition, I built a supporting pipeline around the trading workflow so I could spend less time on repetitive submission steps and more time evaluating how strategy changes affected results.

What I Built

  • Automated uploads of `trader.py` submissions to the IMC platform.
  • Downloaded results, parsed the output, and stored run data in a database.
  • Created a dashboard workflow where new graph views could be added with minimal frontend setup.
  • Built flexible graphing tools to compare data points across runs using different chart types.

Why Build It

  • It reduced repetitive manual work during competition rounds.
  • It made it easier to keep results from different runs organized in one place.
  • It made comparing strategy changes faster, especially when using custom graphs.
